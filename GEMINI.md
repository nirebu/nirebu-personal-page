# Project Context: Nicolò Rebughini's Personal Website

## Overview
This project is the personal website and blog of **Nicolò Rebughini** (nirebu.com). It is a static site generated using **Jekyll** and styled with **Tailwind CSS**.

## Tech Stack
*   **Core:** Jekyll (Ruby Static Site Generator) v4.3+
*   **Styling:** Tailwind CSS v3.3+, PostCSS
*   **Package Managers:**
    *   **Ruby:** Bundler (`Gemfile` which now includes `csv` and `base64` gems for Ruby 3.4.0 compatibility, and a custom plugin for video pagination)
    *   **Node.js:** Yarn/NPM (`package.json`)

## Key Files & Directories
*   **`_config.yml`**: Main configuration file for the Jekyll site (title, url, plugins, etc.). Now includes `youtube_url` for cross-promotion.
*   **`_data/videos.yml`**: Contains structured data for YouTube videos, used for dynamic generation.
*   **`_plugins/video_paginator.rb`**: Custom Jekyll generator plugin for server-side pagination of video listings.
*   **`_posts/`**: Contains blog posts in Markdown format (`YYYY-MM-DD-title.md`).
*   **`_layouts/`**: HTML templates for different page types (`default`, `post`, `home`, etc.). Now includes `video_list.html` for displaying paginated videos.
*   **`_includes/`**: Reusable HTML snippets (header, footer, navbar). `navbar.html` has been modified to link to the new `/videos/` page.
*   **`assets/css/main.scss`**: The main stylesheet entry point. It imports Tailwind directives (`@tailwind base`, etc.) and custom styles.
*   **`package.json`**: Lists Node.js dependencies (Tailwind, Autoprefixer, CSSNano).
*   **`postcss.config.js`**: Configuration for PostCSS, handling Tailwind compilation and minification.
*   **`Gemfile`**: Lists Ruby dependencies (Jekyll and plugins like `jekyll-postcss`, `jekyll-seo-tag`).
*   **`index.md`**: The homepage content, updated with a promotional section for the YouTube channel.
*   **`videos.md`**: This file has been removed, as the video pages are now dynamically generated by the `video_paginator.rb` plugin.

## Setup & Installation

1.  **Install Dependencies:**
    Ensure you have Ruby and Node.js installed. Then run:
    ```bash
    # Install Ruby gems
    bundle install

    # Install Node modules
    yarn install
    # OR
    npm install
    ```

## Development

### Running the Local Server
To start the Jekyll development server with auto-regeneration:
```bash
bundle exec jekyll serve
```
The site will be available at `http://127.0.0.1:4000/` (or the URL specified in the output).

### Building for Production
To build the static site into the `_site` directory:
```bash
JEKYLL_ENV=production bundle exec jekyll build
```
*Note: Setting `JEKYLL_ENV=production` triggers CSS minification via cssnano as configured in `postcss.config.js`.*

## Development Conventions

*   **Content:** Blog posts are written in Markdown and placed in `_posts/`. Video data is managed in `_data/videos.yml`.
*   **Styling:** Use Tailwind utility classes in HTML layouts. Custom CSS rules are defined in `assets/css/main.scss` using `@apply` where necessary.
*   **Asset Management:** The `jekyll-postcss` plugin automatically processes the SCSS/CSS pipeline during the Jekyll build process.
